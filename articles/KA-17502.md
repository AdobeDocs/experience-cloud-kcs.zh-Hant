# AdobeManaged Services (AMS)基本檔案配置

## 說明 {#description}


<b>環境</b>

Adobe Experience Manager

<b>問題/症狀</b>

本檔案將說明AMS標準組態檔案集以及此組態標準背後的想法。


## 解決方法 {#resolution}


<b>預設Enterprise Linux檔案夾結構</b>

在AMS中，基礎安裝使用Enterprise Linux作為基礎作業系統。 安裝Apache Webserver時，它會設定預設的安裝檔案。 以下是透過安裝提供的基本RPM而安裝的預設檔案 `yum` 存放庫。


```
/etc/httpd/
├── conf
│   ├── httpd.conf
│   └── magic
├── conf.d
│   ├── autoindex.conf
│   ├── README
│   ├── userdir.conf
│   └── welcome.conf
├── conf.modules.d
│   ├── 00-base.conf
│   ├── 00-dav.conf
│   ├── 00-lua.conf
│   ├── 00-mpm.conf
│   ├── 00-proxy.conf
│   ├── 00-systemd.conf
│   └── 01-cgi.conf
├── logs - ../../var/log/httpd
├── modules - ../../usr/lib64/httpd/modules
└── run - /run/httpd
```


遵循並遵循安裝設計/結構時，我們將獲得下列優點：

- 更輕鬆支援可預測的版面
- 任何曾經使用Enterprise Linux HTTPD安裝的人都會自動熟悉
- 允許修補作業系統完全支援的週期，不會發生任何衝突或手動調整
- 避免標籤錯誤的檔案前後關聯違反SELinux


注意：

AdobeManaged Services伺服器影像通常有小型作業系統根磁碟機。  我們將資料放入另一個磁碟區中，該磁碟區通常裝載在 `/mnt`

然後我們使用該磁碟區，而不是下列預設目錄的預設值：

<b>檔案根目錄</b>

- 預設： `/var/www/html`
- AMS： `/mnt/var/www/html`


<b>記錄檔目錄</b>

- 預設： `/var/log/httpd`
- AMS： `/mnt/var/log/httpd`


請記住，舊目錄和新目錄會對應回原始掛載點，以避免混淆。 使用個別磁碟區並不重要，但值得注意

<b>AMS附加元件</b>

AMS將新增至Apache Web Server的基礎安裝。

<b>檔案根目錄</b>

AMS預設檔案根：

- 作者： `/mnt/var/www/author/`
- 發佈： `/mnt/var/www/html/`
- 全面收集與健康情況檢查維護： `/mnt/var/www/default/`


<b>暫存與啟用的VirtualHost目錄</b>

下列目錄可讓您建置具有暫存區域的組態檔，如此一來，您便可以處理檔案，並只在檔案準備就緒時啟用它們。

- `/etc/httpd/conf.d/available_vhosts/`
   - 此資料夾會託管您所有名為的VirtualHost /檔案 `.vhost`
- `/etc/httpd/conf.d/enabled_vhosts/`
   - 當您準備好使用時 `.vhost` 您擁有的檔案 `available_vhosts` 資料夾，使用相對路徑將其符號連結至 `enabled_vhosts` 目錄。


<b>其他 `conf.d` 目錄</b>

還有其他在Apache設定中常見的片段。 我們已建立子目錄，允許以簡潔的方式分隔這些檔案，而不會將所有檔案都放在一個目錄中。

1. <b>重寫目錄</b>    此目錄可包含所有 `\*_rewrite.rules` 您建立的檔案，其中包含您的一般 *重寫規則* 與Apache Web伺服器互動的語法 [mod_rewrite](https://httpd.apache.org/docs/current/mod/mod_rewrite.html) 模組。

   - `/etc/httpd/conf.d/rewrites/`
2. <b>白名單目錄</b>    此目錄可包含所有 `\*_whitelist.rules` 您建立的檔案，其中包含您的一般 *IP允許* 或 *需要IP* 與Apache Web伺服器互動的語法 [存取控制](https://httpd.apache.org/docs/2.4/howto/access.html)

   - `/etc/httpd/conf.d/whitelists/`
3. <b>變數目錄</b>    此目錄可包含所有 `\*.vars` 您建立的檔案包含可在設定檔案中使用的變數

   - `/etc/httpd/conf.d/variables/`


<b>Dispatcher模組專屬設定目錄</b>

Apache Web Server極具擴充性，當模組有許多設定檔案時，最佳實務是在安裝基礎目錄下建立自己的設定目錄，而不是將預設目錄雜亂無章。

我們遵循最佳實務，並建立自己的實務。

<b>模組組態檔目錄</b>

- `/etc/httpd/conf.dispatcher.d/`


<b>暫存與啟用的伺服器陣列目錄</b>

下列目錄可讓您建置具有暫存區域的組態檔，如此一來，您便可以處理檔案，並只在檔案準備就緒時啟用它們。

- `/etc/httpd/conf.dispatcher.d/available_farms/`
   - 此資料夾將代管您的所有 `/myfarm {` 已呼叫的檔案 `*_farm.any`
- `/etc/httpd/conf.dispatcher.d/enabled_farms/`
   - 當您準備好使用中的伺服器陣列檔案時 `available_farms` 資料夾，使用相對路徑將其符號連結至 `enabled_farms` 目錄


<b>其他 `conf.dispatcher.d` 目錄</b>

有一些其他片段是Dispatcher陣列檔案設定的子區段，我們建立了子目錄以允許簡潔的方式分隔這些檔案，而不是將所有檔案放在同一個目錄中。

1. <b>快取目錄</b>    此目錄包含所有 `*_cache.any`， `*_invalidate.any` 您建立的檔案包含您想要模組如何處理來自AEM的快取元素以及失效規則語法的規則。  有關本區段的更多詳細資料，請參閱 [此處](https://docs.adobe.com/content/help/zh-Hant/experience-manager-dispatcher/using/configuring/dispatcher-configuration.html#configuring-the-dispatcher-cache-cache)

   - `/etc/httpd/conf.dispatcher.d/cache/`
2. <b>使用者端標頭目錄</b>    此目錄可包含所有 `*_clientheaders.any` 您建立的檔案包含您希望在請求傳入時傳遞給AEM的使用者端標題清單。  有關本區段的更多詳細資料，請參閱 [此處](https://docs.adobe.com/content/help/en/experience-manager-dispatcher/using/configuring/dispatcher-configuration.html#specifying-the-http-headers-to-pass-through-clientheaders)

   - `/etc/httpd/conf.dispatcher.d/clientheaders/`
3. <b>篩選器目錄</b>    此目錄可包含所有 `*_filters.any` 您建立的檔案包含要封鎖或允許流經Dispatcher的流量到達AEM的所有篩選規則。

   - `/etc/httpd/conf.dispatcher.d/filters/`
4. <b>Renders目錄</b>    此目錄可包含所有 `*_renders.any` 您建立的檔案包含Dispatcher將使用內容的每個後端伺服器的連線詳細資料。

   - `/etc/httpd/conf.dispatcher.d/renders/`
5. <b>Vhosts目錄</b>    此目錄可包含所有 `*_vhosts.any` 您建立的檔案包含網域名稱和路徑的清單，以符合特定伺服器陣列與特定後端伺服器的需求。

   - `/etc/httpd/conf.dispatcher.d/vhosts/`

<b>保持理想狀態</b>

Enterprise Linux具有Apache Webserver套裝程式(httpd)的修補週期。

您變更的預設檔案越少安裝越好，原因在於，如果有任何修補的安全性修正或設定改進是透過RPM / Yum命令套用的，則不會在變更的檔案上方套用修正。

而是會建立 `.rpmnew` 檔案。  這表示您會遺漏某些您可能想要的變更，並在設定資料夾中建立更多記憶體。

Enterprise Linux已正確設定，以便以更好的方式處理此使用案例。  它們為您提供可以延伸或覆寫其為您設定的預設值的區域。  在httpd的基本安裝內，您會找到檔案 `/etc/httpd/conf/httpd.conf`，而且其中包含語法，例如：


```
Include conf.modules.d/*.conf
IncludeOptional conf.d/*.conf
```


我們的想法是，Apache希望您在將新檔案新增到時擴充模組和設定 `/etc/httpd/conf.d/` 和 `/etc/httpd/conf.modules.d/` 副檔名為 `.conf`

將Dispatcher模組新增到Apache時，您會建立一個模組，這是完美的範例 `.so` 中的檔案 `/etc/httpd/modules/` 然後在中新增檔案，將其納入 `/etc/httpd/conf.modules.d/02-dispatcher.conf` 包含載入模組的內容 `.so` 檔案。


```
LoadModule dispatcher_module modules/mod_dispatcher.so
```


請注意，我們並未修改Apache提供的任何現有檔案。 我們剛剛將新增到他們原本要前往的目錄。

現在我們在檔案中使用模組 `/etc/httpd/conf.d/dispatcher_vhost.conf` 會初始化模組並載入初始模組特定設定檔。


```
IfModule disp_apache2.c
DispatcherConfig conf.dispatcher.d/dispatcher.any
...SNIP.../IfModule
```


再次強調，您會注意到我們已新增檔案和模組，但未變更任何原始檔案。  這可提供我們所需的功能，並保護我們避免遺失所需的修補程式修正，以及確保套件每次升級時都保持最高相容性。
