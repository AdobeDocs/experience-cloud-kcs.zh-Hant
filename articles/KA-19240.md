# 要刪除的頁面已隱藏，但仍保留在存放庫中

## 說明


在部分情況下，您會發現未在 [!DNL Sites]主控台 (http://localhost:4502/sites.html) 中顯示的頁面，但頁面節點卻在 CRXDE Lite 的存放庫中。此外，該節點會具有 `deleted` 屬性和 `deleted By` 屬性。本文會說明這種情況的原因和解決方法。

<b>環境</b>
AEM as a Cloud Service，AEM 6.5


## 解析度


查找管理員收件匣內「[!UICONTROL 刪除請求]」工作流程的通知，並完成「[!UICONTROL 停用頁面]」和「[!UICONTROL 刪除請求]」的工作流程步驟。

<b>原因</b>
當沒有複寫權限的使用者刪除頁面時，首先該頁面會從 [!DNL Sites] 主控台消失，接著「[!UICONTROL 刪除請求]」工作流程會開始。該頁面將保留在存放庫中，直到工作流完成，如下所示：
1. 沒有複寫權限的使用者刪除頁面。
2. 頁面會從 [!DNL Sites] 主控台 (它會將 `deleted` 屬性和 `deleted By` 屬性設定到頁面節點) 消失。
3. 「[!UICONTROL 刪除請求]」工作流程隨即開始。
4. 管理員群組中的使用者會收到工作流程通知。
5. 管理員群組的使用者之一會完成工作流程步驟。
6. 完成最後一個工作流程步驟會刪除頁面節點。
