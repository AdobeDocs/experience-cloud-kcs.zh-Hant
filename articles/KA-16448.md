# 在AEM例項之間移轉使用者、群組和ACL

## 說明


如何在AEM中使用ACL權限將使用者和群組從一部伺服器移轉至另一部伺服器，或從一部AEM執行個體移轉至另一部。


## 解析度


<b>步驟1:查找管理員和匿名用戶</b>

1. 前往CRXDE lite應用程式/crx/de/index.jsp ，並以管理員使用者身分（在來源系統上）登入。
2. 前往「工具」=「查詢」。
3. 在底部的「查詢」框中，輸入此查詢以查找管理員用戶： */jcr:root/home/users//element(\*,rep:User)@rep:principalName=&quot;admin&quot;。*
4. 按一下「執行」，然後將結果中管理員用戶節點的路徑複製到文本檔案中。
5. 對匿名用戶重複步驟3: */jcr:root/home/users//element(\*,rep:User)@rep:principalName=&quot;anonymous&quot;。*
6. 按一下「執行」，然後將結果中匿名用戶節點的路徑複製到文本檔案中（因此，現在您有兩個路徑，一個用於「管理員」，一個用於「匿名」）。



   例如：

   /home/users/Q/QY5FIMXeQIbGpwZtQ3Dv — 源系統上的管理員用戶

   /home/users/K/Kj1406Qo9IDODc_nk5Ib — 源系統上的匿名用戶


<b>步驟2A:移轉使用者和群組（使用crx2oak或oak-upgrade）</b>

您可以使用在AEM例項之間移轉使用者和群組 [crx2oak](https://helpx.adobe.com/experience-manager/6-4/sites/deploying/using/using-crx2oak.html) 或 [oak-upgrade](https://jackrabbit.apache.org/oak/docs/migration.html) 工具。

1. 下載符合您所使用Oak版本的crx2oak或oak-upgrade jar: 
   1. Oak-upgrade: [https://repo1.maven.org/maven2/org/apache/jackrabbit/oak-upgrade](https://repo1.maven.org/maven2/org/apache/jackrabbit/oak-upgrade)
   2. Crx2Oak: [Maven存放庫](https://mvnrepository.com/artifact/com.adobe.granite/crx2oak?repo=adobe-public)
2. 將jar檔案上傳至AEM伺服器
3. 停止AEM（源實例和目標實例）
4. 在以下命令中替換jar檔案的名稱。
5. 取代 */home/users/Q/QY5FIMXeQIbGpwZtQ3Dv、/home/users/K/Kj1406Qo9IDODc_nk5Ib* 在 *-exclude-paths* 下面命令的part參數，包含來自源系統的管理員和匿名用戶的路徑。
6. 修改線上路徑以符合您的執行個體(新增 *segment-old:* 在路徑之前（如果需要），請參閱此處 [https://jackrabbit.apache.org/oak/docs/migration.html](https://jackrabbit.apache.org/oak/docs/migration.html)): */opt/aem-source/crx-quickstart/repository /opt/aem-destination/crx-quickstart/repository*
7. 然後在伺服器的shell上運行該命令。     `java -Xms2g -Xmx2g -jar oak-upgrade-1.8.12.jar \`

   `--include-paths=/home \`
   `--merge-paths=/home \`
   `--exclude-paths=/home/users/Q/QY5FIMXeQIbGpwZtQ3Dv,/home/users/K/Kj1406Qo9IDODc_nk5Ib,/home/groups/a/administrators,/home/groups/a/analytics-administrators,/home/groups/c/community-moderators,/home/groups/c/content-authors,/home/groups/c/contributor,/home/groups/community/community-groupadmin,/home/groups/community/community-sitecontentmanager,/home/groups/community/community-sitemembers,/home/groups/d/dam-users,/home/groups/default/order-administrators,/home/groups/e/everyone,/home/groups/f/forms-users,/home/groups/forms/fd-administrators,/home/groups/forms/forms-users,/home/groups/geometrixx,/home/groups/media,/home/groups/o/operators,/home/groups/projects,/home/groups/t/tag-administrators,/home/groups/t/target-activity-authors,/home/groups/u/user-administrators,/home/groups/w/workflow-editors,/home/groups/w/workflow-users,/home/users/a/admin,/home/users/a/anonymous,/home/users/mac,/home/users/media,/home/users/projects,/home/users/system,/home/rep:policy,/home/users/rep:policy,/home/groups/rep:policy \`
   `segment-old:/opt/aem-source/crx-quickstart/repository /opt/aem-destination/crx-quickstart/repository  upgradeusers.log &`
8. 完成後，請轉至步驟3以遷移ACL。  如果您無法使用crx2oak進行移轉，請改為遵循下列套件移轉步驟。


<b>步驟2B：移轉使用者和群組（使用套件）</b>

如果使用者未透過LDAP/SAML驗證或CRX2Oak（上述方法A）自動匯入，則您可以封裝使用者和群組。  在此案例中，您會在舊系統上建立兩個不同的套件（排除管理員和匿名現成使用者）。

1. 從上述步驟1（「尋找管理員和匿名使用者」）的結果複製匿名和管理員使用者節點的路徑。

   例如：

   /home/users/Q/QY5FIMXeQIbGpwZtQ3Dv — 我建立包的系統上的管理員用戶

   /home/users/K/Kj1406Qo9IDODc_nk5Ib — 我建立包的系統上的匿名用戶
2. 前往「封裝管理員」， *http://host:port/crx/packmgr/index.jsp*，並以管理員身分登入。
3. 建立包「用戶」
4. 使用下列排除規則將篩選器新增至/home/users的套件設定（位於/home/users篩選器上）:
   1. 排除/home/users/。\*/.tokens
   2. 排除/home/users/Q/QY5FIMXeQIbGpwZtQ3Dv
   3. 排除/home/users/K/Kj1406Qo9IDODc_nk5Ib
   4. 排除/home/users/a/admin
   5. 排除/home/users/a/anonymous
   6. 排除/home/users/system
   7. exclude /home/users/geometrixx
   8. 排除/home/users/media
   9. 排除/home/users/projects
   10. 排除/home/users/mac
5. 建立套件。
6. 下載套件。
7. 將電腦上的封裝zip檔案解壓縮： `jar -xvf users.zip META-INF/vault/filter.xml`
8. 在文字編輯器中開啟檔案META-INF/vault/filter.xml 。
9. 將mode=&quot;merge&quot;新增至篩選器……標籤，例如：






   ```
   ?xml version="1.0" encoding="UTF-8"?    workspaceFilter version="1.0"    filter root="/home/users" mode="merge"    exclude pattern="/home/users/.*/.tokens"/    exclude pattern="/home/users/Q/QY5FIMXeQIbGpwZtQ3Dv"/    exclude pattern="/home/users/K/Kj1406Qo9IDODc_nk5Ib"/    exclude pattern="/home/users/a/admin"/    exclude pattern="/home/users/a/anonymous"/    exclude pattern="/home/users/system"/    exclude pattern="/home/users/geometrixx"/    exclude pattern="/home/users/media"/    exclude pattern="/home/users/projects"/    exclude pattern="/home/users/mac"/    /filter    /workspaceFilter
   ```




10. 將修改的套件內容重新壓縮，使其包含變更。

   `jar -uvf users.zip META-INF/vault/filter.xml`
11. 建立包含篩選規則/home/groups的「群組」套件。
12. 對組包重複步驟11-14。
13. （僅限升級）如果執行移轉至較新的AEM版本，請安裝最新的本機AEM例項 *舊版*（使用nosamplecontent），然後安裝使用者套件，然後再安裝該群組套件。 然後，在該執行個體上執行到新版本的就地升級。 這會將使用者轉換為新的Oak表示法。 就地升級後，重新封裝使用者，將其連接至您打算升級的執行個體。 對使用者群組執行相同操作。
14. 在新系統上安裝用戶包。
15. 在新系統上安裝組包。
16. 如果您要從舊版AEM移轉至6.3，請前往/useradmin UI，並將使用者復寫接收器新增至「administrators」群組。




<b>步驟 3.遷移ACL</b>

如果您能將工具（ACS公域）安裝到AEM，請執行下列步驟：

1. 下載並安裝ACS Commons。
2. 按照此處提供的步驟建立ACL包。
3. 前往http://aem-host:port/crx/packmgr/index.jsp and以管理員身分登入。
4. 按一下ACL包。
5. 按一下編輯。
6. 選取「進階」標籤（請參閱下方的螢幕擷圖）。
7. 在「AC處理」下拉菜單中，選擇「合併」以避免刪除目標系統上的現有ACL。 在AEM的不同版本之間遷移ACL時，這一點尤其重要，因為它可以避免移除現成可用的ACL。


如果您 *not* 可將工具（ACS公域）安裝到AEM，然後遵循這些步驟。 請注意，您執行這些命令的電腦必須是Mac OS、Linux或Windows（使用Cygwin），且已安裝cURL、python和Java SDK。

1. 前往http://src-aem-host:port/crx/packmgr/index.jsp and以管理員身分登入。
2. 建立名為「ACL-migration」的包
3. 按一下「編輯」按鈕。
4. 選擇「高級」頁簽，並將「交流處理模式」設定為「合併」。
5. 儲存。
6. 建置套件並下載。
7. 在檔案系統上，對包運行此命令以解壓META-INF/vault/filter.xml檔案： 

   `jar -xvf ACL-migration-1.0.zip META-INF/vault/filter.xml`


8. 在同一目錄下，運行此命令從源實例/content下下載ACL路徑的json檔案（設定用戶名和密碼並正確主機）: 

   `curl -u admin:admin 'http://aemhost/crx/de/query.jsp?' -G --data-urlencode '_dc=1507011481908&_charset_=utf-8&type=xpath&stmt=/jcr:root/content//element(*,rep:ACL)&showResults=true'  data.json`


9. 建立檔案generate-package-filter.py，並將python代碼貼到其下： 

   `import` `json`


   `from pprint import` `pprint`





   `with open` `(` `'data.json') as data_file:`


   `    data = json.load(data_file)`





   `print("?xml version=\"1.0\" encoding=\"UTF-8\"?")`


   `print(` `"workspaceFilter version=\"1.0\""` `)`


   `for` `item ` `in` `data` `"results"` `:`


   `    ` `print(` `"filter root=\"{path}\" /"` `.` `format` `(path=item` `"path"` `))`


   `print(` `"/workspaceFilter"` `)`
10. 從建立data.json的相同資料夾執行python指令碼，並將輸出儲存至META-INF/vault/filter.xml（取代filter.xml的現有內容）: 

   `python generate-packge-filter.py  META-INF/vault/filter.xml`


11. 使用以下命令更新zip檔案中的filter.xml:

   `jar -uvf ACL-migration-1.0.zip META-INF/vault/filter.xml`


12. 將zip檔案上傳至來源執行個體套件管理器： http://src-aem-host:port/crx/packmgr/index.jsp
13. 按一下「建置」或「重建」以建置套件。
14. 從來源AEM伺服器下載套件。
15. 將套件上傳至目標AEM伺服器的套件管理器： http://dst-aem-host:port/crx/packmgr/index.jsp
16. 按一下「安裝」以安裝。
17. 對於更改路徑curl命令的任何其他路徑，重複步驟8-16。 例如，這會在/etc（而非/content）底下取得ACL: 

   | <br><br><br>`curl -u admin:admin 'http://aemhost/crx/de/query.jsp?' -G --data-urlencode '_dc=1507011481908&_charset_=utf-8&type=xpath&stmt=/jcr:root/etc//element(*,rep:ACL)&showResults=true'  data.json` |
