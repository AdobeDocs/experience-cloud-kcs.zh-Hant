# 轉出失敗，因為發生衝突mixin（和使用ClassicUI）

## 說明


由於某些衝突，頁面轉出失敗 *jcr:mixinTypes* 在頁面的縮圖節點上。

使用縮圖設定頁面時會發生衝突 <b>使用ClassicUI</b> 然後開始推出（第一個推出作品）。

此時，轉出主版頁面失敗，因為頁面縮圖結構上的mixin在以下兩者之間發生衝突：

- dam：縮圖
- cq:LiveRelationship

<b>要再現的步驟：</b>

1. 啟動vanilla AEM 6.5.13.0
2. 轉到 <b>/siteadmin</b> (因此， <b>傳統UI</b>)
3. 開啟的頁面屬性 */content/we-retail/language-master/en/women*
4. 在影像中 *標籤* 將任何影像設為縮圖
5. 轉出頁面
6. 現在，請前往 <b>Sites UI</b>
7. 開啟的頁面屬性 */content/we-retail/us/en/women* an
8. 在 *縮圖* 按一下 *產生預覽* 並儲存變更
9. 現在在/siteadmin中重新轉出頁面



## 解析度


轉出期間發生此問題，因為AEM無法為即時副本頁面(/content/we-retail/us/en/women/jcr:content/image/file/jcr:content)在jcr:content下建立「dam:thumbnails」節點，而轉出期間原因為mixinType「dam:Thumbnails」未套用至即時副本頁面節點「/content/we-retail/us/en/jcr:content/image/file/jcr:content」和「dam:thumbntabl」資料夾，只能在「dam:thumbnail」下建立。

MSM預設行為是在轉出0期間忽略新增至來源節點的混合，因此不會新增至「/content/we-retail/us/en/women/jcr:content/image/file/jcr:content」。

為了修正此問題，可透過內容更新動作設定來變更此行為。
預設會忽略每個「混合類型」，如下方螢幕擷取所示：

![](assets/6e22f175-e313-ed11-b83d-002248086a27.png)

應更新以使用：


```
(?!.*dam:Thumbnails).*
```


如下方螢幕擷取所示：

![](assets/4d5f7db7-e313-ed11-b83d-002248086a27.png)

這應該可以解決問題。
