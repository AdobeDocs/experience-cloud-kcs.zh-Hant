# 如何最佳化Oak Lucene索引以縮短索引時間

## 說明


最佳化AEM Oak lucene索引以縮短重新索引時間。
<br><br>環境<br><br>
AEM 6.0 + Oak 1.0.16或更新版本AEM 6.1和AEM 6.2


## 解析度


您可以微調oak: QueryIndexDefinition並搭配「includedPaths」、「excludedPaths」、「name」和「indexPath」等屬性，以最佳化Oak Lucene索引。

- &quot;includedPaths&quot;和&quot;excludedPaths&quot;屬性會限制索引在重新索引時遍歷的路徑，以及它監聽的索引更新路徑。
- 「name」和「indexPath」屬性使索引日誌更易於理解，因為它會用索引的路徑或名稱標籤日誌消息。  例如:



| `30.01.2017 14:07:12.934 *DEBUG* pool-8-thread-2 org.apache.jackrabbit.oak.plugins.index.lucene.LuceneIndexEditor damassetlucene(` `/oak` `:index` `/damAssetLucene` `) = Indexed 9000 nodes...` |
| --- |


<b>最佳化索引的步驟</b>:

1. 前往http://*aem-host:port*/crx/de/index.jsp登入並擔任管理員
2. 瀏覽至/oak:index
3. 將下面突出顯示的屬性添加到相應的索引。  此 <b>includedPaths</b> 和 <b>excludedPaths</b> 屬性應為多值字串（字串）
4. 按一下「[!UICONTROL 全部儲存]&quot;在CRXDe中
