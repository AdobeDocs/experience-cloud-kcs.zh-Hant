# AEP — 如果資料集未啟用 [!DNL Profile]，這會對區段有何影響？

## 說明

如果資料集未啟用 [!DNL Profile]，該資料集中包含的資料是否未視為分段資格（即使該資料集/結構中已指定身分）?

## 解析度

不，這根本沒有被考慮，這是Experience Platform的運作方式。 分段適用於UPS（統一設定檔服務）層級，而非資料湖層級。

若 [!DNL Profile] 未啟用，則進入資料湖的資料不會升級至統一的設定檔和身分服務
