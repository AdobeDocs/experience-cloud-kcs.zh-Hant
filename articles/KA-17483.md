# 只讀檔案

## 說明

文檔將說明哪些檔案被鎖定在不要更改的中，以及如何正確進行所需的配置設定。<br><br> <br><br><br><br><br>原因為何?<br><br><br><br> <br><br>
當AMS調配系統時，會推出基線配置，使所有功能都可正常運作且安全。  AMS想要保證這些不是功能和安全性的基準。  要完成此操作，某些檔案被標籤為只讀和不可變，以避免您更改它們。

版面不會防止您變更其行為並覆寫您需要的任何變更。  與其更改這些檔案，您將覆蓋您自己的檔案，以取代原始檔案。

這也可讓您放心，當AMS修補Dispatcher，並附上最新修正和安全性增強功能，且不會變更您的檔案。  接著，您可以繼續從這些改善中獲益，並只採用您想要的變更。
![展示一個保齡球道，球滾著。  球上有一支箭，箭上寫著「你」。  水溝阻擋器被抬起，上面有「不可修改的檔案」。](https://helpx.adobe.com/content/dam/kb/en/ams-dispatcher-manual/bowling-file-immutability.png "bowling-file-micutibility")<br><br><br><br> <br><br>
如上圖所示，不可修改的檔案不會阻止您玩遊戲。  他們只是阻止你傷害你的表演，讓你留在巷子裡。  此方法允許我們使用幾個非常關鍵的功能：

- 自訂會在自己的安全空間中處理
- 自訂變更的覆蓋反映AEM中覆蓋方法的覆蓋
- 修補AMS設定可以完成，而無須更改自定義
- 測試基本安裝與自訂設定可同時完成，以幫助識別問題是否是由自訂或其他原因造成

## 解析度

<br><br>如何進行變更<br><br><br><br> <br><br>變數<br><br><br><br> <br><br>
變數可讓您進行功能變更，而無須變更設定檔案本身。  設定的某些元素可透過調整變數的值來調整。  可從檔案中反白顯示的範例 *`/etc/httpd/conf.d/dispatcher_vhost.conf`* 如下所示：
<br><br><br><br><br> <br><br><br><br>

`Include /etc/httpd/conf.d/variables/ams_default.vars IfModule disp_apache2.c    DispatcherConfig conf.dispatcher.d/dispatcher.any    DispatcherLog    logs/dispatcher.log    DispatcherLogLevel ${DISP_LOG_LEVEL}    DispatcherDeclineRoot 0    DispatcherUseProcessedURL 1/IfModule`

<br><br><br><br><br> <br><br>
了解 <b>`DispatcherLogLevel`</b> 指令具有的變數為 `DISP_LOG_LEVEL` 而不是你在那裡看到的正常值。  在程式碼的該節上方，您也會看到變數檔案的include陳述式。  變數檔案*`/etc/httpd/conf.d/variables/ams_default.vars`*是我們下一步想看的地方。  以下是變數檔案的內容：
<br><br><br><br><br> <br><br><br><br>

`Define DISP_LOG_LEVEL info Define AUTHOR_WHITELIST_ENABLED 0 Define PUBLISH_WHITELIST_ENABLED 0 Define LIVECYCLE_WHITELIST_ENABLED 0 Define AUTHOR_FORCE_SSL 1 Define PUBLISH_FORCE_SSL 0 Define LIVECYCLE_FORCE_SSL 1 `

<br><br><br><br><br> <br><br>
如果您看到 `DISP_LOG_LEVEL` 變數為資訊。  我們可以調整以追蹤、除錯，或您選擇的數字值/層級。  現在，每個控制日誌級別的位置都會自動調整。
<br><br><br><br> <br><br>覆蓋方法<br><br><br><br> <br><br>
請了解 [頂級包含檔案](https://helpx.adobe.com/experience-manager/kb/ams-dispatcher-manual/explanation-config-files.html#include-order) 因為這些將是您進行任何自訂的起點。  首先，我們有一個案例，想要新增要指向此Dispatcher的新網域名稱。  我們將使用的網域範例為 <b>`we-retail.adobe.com`. </b> 首先，我們會將現有的設定檔案複製到新檔案，以便新增變更：
<br><br><br><br><br> <br><br><br><br>

`$ cp /etc/httpd/conf .d /available_vhosts/aem_publish .vhost /etc/httpd/conf .d /available_vhosts/weretail_publish .vhost`

<br><br><br><br><br> <br><br>
我們複製了 *`aem_publish.vhost`* 因為檔案已經具備了讓事情正常運作所需的內容，而且我們不想重新創造一個本已強大的開端。  現在，我們編輯 *`weretail.vhost`* 檔案，並進行所需的變更。

變更前:
<br><br><br><br><br> <br><br><br><br>

`VirtualHost *:80      ServerName    publish      ServerAlias    ${PUBLISH_DEFAULT_HOSTNAME}      DocumentRoot    ${PUBLISH_DOCROOT}      IfModule mod_headers.c              Header always add X-Dispatcher ${DISP_ID}              Header always add X-Vhost "publish"              Header merge X-Frame-Options SAMEORIGIN "expr=%{resp:X-Frame-Options}!='SAMEORIGIN'"              Header merge X-Content-Type-Options nosniff "expr=%{resp:X-Content-Type-Options}!='nosniff'"          Header append Vary User-Agent env=!dont-vary      /IfModule ....... SNIP....... /VirtualHost`

<br><br><br><br><br> <br><br>
之後：
<br><br><br><br><br> <br><br><br><br>

`VirtualHost *:80      ServerName    weretail-publish      ServerAlias    we-retail.adobe.com      DocumentRoot    ${PUBLISH_DOCROOT}      IfModule mod_headers.c              Header always add X-Dispatcher ${DISP_ID}              Header always add X-Vhost "werteail-publish"              Header merge X-Frame-Options SAMEORIGIN "expr=%{resp:X-Frame-Options}!='SAMEORIGIN'"              Header merge X-Content-Type-Options nosniff "expr=%{resp:X-Content-Type-Options}!='nosniff'"          Header append Vary User-Agent env=!dont-vary      /IfModule ....... SNIP....... /VirtualHost`

<br><br><br><br><br> <br><br>
現在，我們已更新 `ServerName` 和 `ServerAlias` 以符合新網域名稱，以及更新其他階層連結標題。  現在啟用新檔案，讓apache知道可以使用新檔案：
<br><br><br><br><br> <br><br><br><br>

`$ cd /etc/httpd/conf .d /enabled_vhosts/ ; ln -s .. /available_vhosts/weretail_publish .vhost`

<br><br><br><br><br> <br><br>
現在，Apache webserver知道網域是某些內容應會產生的流量，但我們仍需通知Dispatcher模組，它有新的網域名稱需遵守。  首先，我們將建立新的\*`_vhost.any` 檔案 <b>`/etc/httpd/conf.dispatcher.d/vhosts/weretail_vhosts.any`</b> 在檔案里，我們要放入我們要遵守的域名：
<br><br><br><br><br> <br><br><br><br>

`we-retail.adobe.com`

<br><br><br><br><br> <br><br>
現在，我們需要製作新的伺服器陣列檔案，以使用我們的 `vhost` 進入檔案，我們首先將一個強啟動檔案複製到自己的新啟動檔案。
<br><br><br><br><br> <br><br><br><br>

`$ cp /etc/httpd/conf .dispatcher.d /available_farms/999_ams_publish_farm .any /etc/httpd/conf .dispatcher.d /available_farms/400_weretail_publish_farm .any`

<br><br><br><br><br> <br><br>
讓我們顯示對此伺服器陣列檔案所需的變更

變更前:
<br><br><br><br><br> <br><br><br><br>

`/publishfarm {       /virtualhosts {          $include "/etc/httpd/conf.dispatcher.d/vhosts/ams_publish_vhosts.any"      } ........SNIP......... }`

<br><br><br><br><br> <br><br>
之後：
<br><br><br><br><br> <br><br><br><br>

`/weretailpublishfarm {       /virtualhosts {          $include "/etc/httpd/conf.dispatcher.d/vhosts/weretail_publish_vhosts.any"      } ........SNIP......... }`

<br><br><br><br><br> <br><br>
現在，我們已更新伺服器陣列名稱，並將其用於 `/virtualhosts` 伺服器陣列設定的區段。  我們需要啟用此新伺服器陣列檔案，以便它能在執行中的設定中使用：
<br><br><br><br><br> <br><br><br><br>

`$ cd /etc/httpd/conf .dispatcher.d /enabled_farms/ ; ln -s .. /available_farms/400_weretail_publish_farm .any`


現在，我們只是重新載入Web伺服器服務並使用新域！

注意:

請注意，我們只變更了變更所需的片段，並運用基線組態檔隨附的現有包含和程式碼。  我們只需要勾勒出我們需要改變的要素。  讓事情變得簡單多了，而且讓我們能夠維護更少的代碼
