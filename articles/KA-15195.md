# [!DNL Target]:根據建議設計得出的建議結果

## 說明

<br><br><br>環境<br><br>
Target


<br><br>問題/症狀<br><br>
使用者已建立建議活動，在條件結果結束後，他們決定變更建議輪播的設計。



條件中沒有變更。 為什麼他們需要等待結果就緒，只為設計變更？


## 解析度


建議引擎根據設計準備Recs候選項的數量。 因此，對於具有三個插槽的設計，Recs算法將嘗試生成三個可選設計插槽。

因此，當設計改變時，它需要重新計算候選項的數量。 這就是為什麼在對設計進行變更時需要再次等待結果。
