# 從JVM進行對話串傾印

## 說明 {#description}


### <b>環境</b>

Experience Manager



### <b>問題/症狀</b>

如何在Linux、UNIX或Windows上從JVM進行對話串傾印？

對話串傾印是目前在Java虛擬機器器(JVM)中作用中的所有Java對話串的清單。

有幾種方法可以從JVM進行對話串傾印。 強烈建議進行1個以上的對話串傾印。 好的實務是定期進行10個對話串傾印（例如，每10秒傾印一個對話串）。


## 解決方法 {#resolution}


步驟1：取得Java程式的PID



若要取得對話串傾印，您必須先取得Java流程的PID資訊。

Java JDK隨附jps命令，列出所有Java程式ID。 您可以執行此命令，如下所示：


```
jps -l 70660 sun.tools.jps.Jps 70305
```


<b>注意：</b> 在Linux和UNIX中，您可能必須執行此命令為 <b>sudo -u使用者jps -l</b>，其中「user」是執行Java程式之使用者的使用者名稱。

如果這樣行不通，或您仍然找不到Java程式，（路徑未設定、未安裝JDK或較舊的Java版本），請使用

- UNIX、Linux和Mac OS X： <b>ps -el | grep java</b>
- Windows：按Ctrl+Shift+Esc開啟工作管理員並找到Java程式的PID


步驟2：向JVM請求對話串傾印

<b>jstack</b>

如果已安裝/可用，我們建議使用 <b>jstack</b> 工具。 它會將對話串傾印到命令列主控台。

若要使用jstack取得對話串傾印，請執行以下命令：
<b>jstack -l `<` pid`>` </b>

您可以使用主控台輸出重新導向/附加指令，將連續的執行緒傾印輸出至檔案：
<b>jstack -l `<` pid`>`  `>` `>`  threaddumps.log</b>

附註:

- jstack工具自JDK 1.5起便已推出（僅適用於Windows上的JVM，有些JDK 1.5和JDK 1.6版本才有提供）。
- jstack可運作，即使 `-Xrs` jvm引數已啟用
- 無法使用JDK 1.6中的jstack工具從JDK 1.5上執行的程式取得執行緒傾印。
- 在Linux和UNIX中，您需要以擁有Java流程的使用者身分執行命令：

  <b>須藤 — u *java-user* jstack -l `<` *pid`>`</b>*  

  (`<` java-user`>`  應取代為Java程式執行身分的使用者id)
- 在Windows中，如果您執行jstack並收到「沒有足夠的可用儲存空間來處理此命令」錯誤，則您必須以Windows SYSTEM使用者或擁有Java處理序的使用者身分執行jstack。  您可以使用psexec來執行此操作，您可下載該工具 [此處](https://technet.microsoft.com/en-us/sysinternals/bb897553.aspx). 若要以SYSTEM使用者身分執行jstack，請使用如下命令：

  <b>psexec -s jstack `<` *pid`>`*  `>` `>`  threaddumps.log</b>

  如果您無法在伺服器上安裝psexec，則可以建立包含命令的.bat檔案並執行它 [使用Windows工作排程器（以不同使用者的身分）](https://technet.microsoft.com/en-us/library/cc748993%28v=ws.11%29.aspx).
- 如果Java程式沒有回應，則使用選項有時會有幫助 <b>-J-d64</b> （在64位元系統上），例如：

  <b>jstack -J-d64 -l *`<`pid`>`* `>` `>`  threaddumps.log</b>
- 如果jstack命令(<b>jstack -l `<` pid`>`  `>` `>`  threaddumps.log</b>)擲回錯誤 `[` 1`]`  在底下，然後以擁有java流程的使用者身分執行命令。  例如:

  sudo -u sling jstack -l `<` pid`>`  `>` `>`  threaddumps.log

JSTACK指令碼

以下是 [指令碼](https://github.com/cqsupport/jstackSeries.sh/blob/master/jstackSeries.sh) (改編自 [eclipse.org](http://wiki.eclipse.org/How_to_report_a_deadlock#jstackSeries_--_jstack_sampling_in_fixed_time_intervals_.28tested_on_Linux.29))進行一連串對話串傾印。  它也會使用top命令來使用執行緒層級cpu使用量。

請像這樣執行：
<b>須藤 — u *`<` 使用者`>`*jstackSeries.sh `<` *pid`>`* `<` *aemserveruser`>`* `<` *count`>`* `<` *delay`>`</b>*

例如：
<b>sudo -u aemuser jstackSeries.sh 1234 aemserveruser 10 3</b>

- <b>1234</b> 是Java程式的pid
- <b>cq5serveruser</b> 是Java處理序執行的Linux或UNIX使用者
- <b>10</b> 是要進行多少對話串傾印
- <b>3</b> 是每個傾印之間的延遲


<b>注意： </b>頂端輸出具有十進位格式的原生執行緒ID，而jstack輸出具有十六進位格式的ID。  您可以將執行緒ID轉換為十六進位，以比對從頂端輸出到jstack輸出的高cpu執行緒。

除了上述指令碼外，我們也有類似的 [Windows Powershell指令碼和github上的AdobeAEM特定指令碼](https://github.com/cqsupport/jstackSeries.sh).

### <b>取得對話串傾印的替代方法</b>

如果 <b>jstack</b> 工具無法供您使用，然後您可以依照下列步驟進行對話串傾印：

<b>注意：</b> 如果commandline引數，則某些工具無法從JVM進行對話串傾印 `-Xrs` 已啟用。 如果您在進行對話串傾印時遇到問題，請檢查此選項是否已啟用。

### UNIX、Mac OS X和Linux （JDK 1.4或更低版本）

在UNIX、Mac OS X和Linux上，您可以傳送QUIT訊號給Java程式，以指示它輸出執行緒傾印到標準輸出。

1. 執行此命令以執行此操作：

   <b>kill -QUIT *`<`pid`>`</b>*

   您可能需要執行此命令 <b>sudo -u使用者終止 — 結束 *`<`pid`>`</b>* 其中，「user」是執行Java流程的使用者。
2. 如果您使用啟動CQSE <b>crx-quickstart/server/start</b> 指令碼，然後您的對話串傾印將會輸出到 <b>crx-quickstart/server/logs/startup.log</b>. 如果您使用協力廠商應用程式伺服器，例如JBoss、WebSphere、Tomcat或其他，請參閱伺服器的檔案，找出將標準輸出導向哪個檔案。


### Windows:

### JDK 1.X

1. 下載javadump.exe （附於下方）。
2. 使用以下三個引數啟動JVM （它們的順序必須正確）：

   <b>-XX：+UnlockDiagnosticVMOptions -XX：+LogVMOutput -XX：LogFile=*C:\temp\jvmoutput.log</b>*
3. 按Ctrl+Shift+Esc開啟「工作管理員」。
4. 尋找Java程式的PID。
5. 從命令列，執行

   <b>javadump.exe *`<`pid`>`</b>*
6. 對話串傾印將顯示在 `jvmoutput.log` 步驟2中提到的檔案。


### JDK 1.6

從取得對話串傾印 <b>jconsole</b> 工具，使用外掛程式： `[` 0`]`

以下說明如何請求對話串傾印：

1. 將下列引數新增至執行Communique的jvm ： <b>-Dcom.sun.management.jmxremote</b>
2. 下載並安裝JDK 1.6 （如果尚未完成）。
3. 下載並解壓縮 [執行緒傾印分析器公用程式](https://github.com/irockel/tda). `[` 1`]`
4. 執行 `jconsole.exe` （JDK 1.6版）：

                               `jconsole.exe -pluginpath /path/to/file/tda.jar`
5. 按一下 <b>執行緒傾印</b> 標籤。
6. 按一下 <b>請求對話串傾印</b> 連結。


<b>注意：</b> 如果您正在執行AEM 6。\*並想觀察執行中的對話串，您可以要求 `http://`&lt;` host`>` :`&lt;` port`>` /system/console/status-Threads` 以取得執行緒清單。 不過，請注意，這些對話串傾印無法用於Samurai或tda等對話串傾印分析工具。
